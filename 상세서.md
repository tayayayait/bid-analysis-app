## 1. 디자인 시스템 개요

### 1.1 디자인 철학
- **데이터 중심(Data-First)**: 복잡한 입찰 데이터를 직관적으로 시각화
- **기능 우선(Function-First)**: 화려함보다 실용성과 가독성 중시
- **전문가 친화적(Expert-Friendly)**: 입찰 담당자가 빠르게 의사결정할 수 있는 정보 구조

### 1.2 기술 스택
| 구분 | 선택 기술 | 비고 |
|------|----------|------|
| 플랫폼 | Electron + React (또는 Tauri + React) | Windows Desktop App |
| 차트 라이브러리 | Recharts / Chart.js | 분포도, 히스토그램 지원 |
| UI 프레임워크 | 자체 디자인 시스템 (Vanilla CSS) | 커스텀 컴포넌트 |
| 상태 관리 | Zustand / React Query | API 캐싱 및 상태 |
| 데이터베이스 | SQLite (Local) | 로컬 DB 적재 |

| 속성 | 값 |
|------|-----|

```

│  📊 활성 공고 수                │ ← 아이콘 + 레이블 (text-caption)
│                                 │
```

| 속성 | 값 |
|------|-----|
| 너비 | 유동적 (Grid 기반), 최소 200px |
| 높이 | 120px ~ 140px |
| 패딩 | `20px` |

## 10. API 데이터 활용 매핑

### 10.1 화면별 API 매핑

| 화면 | 사용 API 서비스 | 주요 오퍼레이션 |
|------|----------------|-----------------|
| 대시보드 | 입찰공고정보서비스, 낙찰정보서비스 | `getBidPblancListInfo*`, `getScsbidListSttus*` |
| 입찰 공고 목록 | 입찰공고정보서비스 | `getBidPblancListInfoCnstwk`, `getBidPblancListInfoServc` 등 |
| 공고 상세 | 입찰공고정보서비스 | `getBidPblancListInfoCnstwkBsisAmount` (기초금액 포함) |
| 낙찰 현황 | 낙찰정보서비스 | `getScsbidListSttus*`, `getOpengResultListInfo*` |
| 개찰 결과 상세 | 낙찰정보서비스 | `getOpengResultListInfoOpengCompt`, `*PreparPcDetail` (예비가격) |
| 경쟁사 분석 | 사용자정보서비스, 낙찰정보서비스 | `getPrcrmntCorpBasicInfo02`, 낙찰 이력 조회 |
| 투찰 예측 | 전체 API 종합 분석 | 과거 데이터 기반 ML 모델 입력 |

### 10.2 주요 데이터 필드 시각화 매핑

| 데이터 필드 | 시각화 컴포넌트 | 설명 |
|-------------|-----------------|------|
| `sucsfbidRate` (낙찰률) | 히스토그램, 박스플롯 | 투찰률 분포 시각화 |
| `prtcptCnum` (참가업체수) | 막대 차트, KPI 카드 | 경쟁 강도 표시 |
| `bidClseDt`, `opengDt` | 타임라인, 캘린더 | 일정 관리 |
| `rlOpengDt` (실개찰일시) | 시계열 차트 | 시간대별 패턴 분석 |
| `bidwinnrBizno` (업체 사업자번호) | 링크, 필터 | 업체별 이력 추적 |
